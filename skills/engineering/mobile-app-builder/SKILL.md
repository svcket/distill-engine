# Mobile App Builder

## Purpose

Specializing in cross-platform and native mobile application development.

## When Invoked

Invoke when:
- Designing and implementing mobile user interfaces
- Building mobile-first features and functionalities
- Optimizing mobile app performance and battery life
- Ensuring platform-specific compliance and guidelines
- Integrating with mobile hardware and native APIs
- Managing app distribution and submission processes

## Inputs

- Mobile-specific design mockups (iOS/Android)
- Target platform requirements (Native/Flutter/React Native)
- Feature specifications and user stories
- Performance and battery usage targets
- App store submission requirements and guidelines

## Process

### 1. Mobile Architecture and Setup
- Choose appropriate mobile development framework
- Set up development environment and platform-specific SDKs
- Design mobile-first navigation and state management
- Establish automated testing and build processes

### 2. UI/UX Implementation
- Build responsive and fluid mobile interfaces
- Implement platform-specific gestures and animations
- Ensure accessibility compliance (VoiceOver/TalkBack)
- Optimize assets for various screen sizes and densities

### 3. Feature Development and Integration
- Implement core features using mobile-specific APIs
- Integrate with hardware (Camera, GPS, Biometrics)
- Manage offline data and local storage
- Ensure secure communication with backend services

### 4. Performance and Quality Assurance
- Optimize app launch time and memory usage
- Conduct battery life and network efficiency tests
- Perform extensive device and OS version testing
- Manage app signing, distribution, and store submission

## Outputs

- High-quality, performant mobile application (IPA/APK)
- Reusable mobile component library
- Mobile performance and battery usage reports
- App store submission assets and metadata
- Mobile-specific unit and E2E tests

## Quality Bar

- App launch time < 2 seconds
- Smooth 60 FPS animations and interactions
- < 100MB initial download size
- 99%+ crash-free session rate
- Full compliance with App Store/Play Store guidelines

## Dependencies

- Mobile frameworks (React Native, Flutter, Swift, Kotlin)
- Mobile-specific testing tools (Appium, Detox)
- Distribution platforms (TestFlight, Firebase App Distribution)
- Native mobile development environments (Xcode, Android Studio)

## Failure Modes

- Frequent app crashes and slow performance
- Poor battery usage and high resource consumption
- Inconsistent behavior across different devices and OS versions
- Rejection from app stores due to non-compliance
- Inaccessible or frustrating mobile user experiences

## Escalation / Recovery

If crash rates increase, prioritize identifying and fixing the root cause. If performance targets are missed, profile the app and optimize resource-heavy components.
